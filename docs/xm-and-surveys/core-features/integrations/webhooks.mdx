### **Webhook Triggers**

Webhooks are configured to send notifications based on trigger events. The available triggers include:

- **`responseCreated`**

- **`responseUpdated`**

- **`responseFinished`**

---

## **Creating Webhooks**

You can create webhooks either through the **Formbricks App UI** or programmatically via the **Webhook API**.

## **Creating Webhooks via UI**

- **Log in to Formbricks**
  and click on the `Configuration` tab in the left sidebar and then click on the `Integrations` tab.

![Step one](/images/xm-and-surveys/core-features/integrations/webhooks/integrations-tab.webp)

- Click on **Manage Webhooks** & then **Add Webhook** button:

![Step two](https://res.cloudinary.com/dwdb9tvii/image/upload/v1738094259/j4a92z2q43twgamogpny.webp)

- Add your webhook listener endpoint & test it to make sure it can receive the test endpoint otherwise you will not be able to save it.

![Step three](https://res.cloudinary.com/dwdb9tvii/image/upload/v1738094617/image_kubsnz.jpg)

- Now add the triggers you want to listen to and the surveys!

- That’s it! Your webhooks will not start receiving data as soon as it arrives!

![Step five](https://res.cloudinary.com/dwdb9tvii/image/upload/v1738094816/image_xvrel1.jpg)

## **Creating Webhooks via API**

Use our documented methods on the **Creation**, **List**, and **Deletion** endpoints of the Webhook API mentioned in our [API v2 playground](https://formbricks.com/docs/api-v2-reference/management-api-%3E-webhooks/get-webhooks).

If you encounter any issues or need help setting up webhooks, feel free to reach out to us on [GitHub Discussions](https://github.com/formbricks/formbricks/discussions). 😃

## Contact Attributes in Webhooks

When a response is linked to an identified contact (user), the webhook payload includes `contact` and `contactAttributes` fields. Contact attributes are key-value pairs that store information about your users, such as:

- User details: name, email, company
- Plan information: subscription tier, billing status
- Custom attributes: role, department, custom fields

The `contactAttributes` field contains a snapshot of the contact's attributes at the time the response was created. This allows you to:
- Segment responses by user attributes
- Personalize follow-up actions based on user data
- Sync response data with other tools using contact information

<Note>
  For anonymous responses (link surveys without user identification), both `contact` and `contactAttributes` will be `null`.
</Note>

## Using Contact Attributes in Integrations

Contact attributes enable powerful workflows by providing context about who submitted the response. Here are some common use cases:

### Accessing Contact Attributes

In webhook payloads, contact attributes are available under `data.contactAttributes`. You can access individual attributes like this:

- **Email**: `data.contactAttributes.email`
- **Name**: `data.contactAttributes.name`
- **Plan**: `data.contactAttributes.plan`
- **Company**: `data.contactAttributes.company`
- **Custom attributes**: `data.contactAttributes.yourCustomAttribute`

### Example Use Cases

**1. Segment by Plan Type**
- Route Pro plan users to a priority support channel
- Send different follow-up emails based on subscription tier

**2. Personalize Responses**
- Address users by name in automated messages
- Include company information in CRM updates

**3. Enrich Analytics**
- Track satisfaction scores by user role or department
- Analyze feedback patterns across different customer segments

**4. Conditional Workflows**
- Trigger different actions based on user attributes
- Filter responses by specific attribute values

### Integration Examples

**Zapier / Make.com**

When setting up your integration, you can access contact attributes in the data mapping:

1. Select the webhook trigger
2. Use the test response to see available fields
3. Map `data.contactAttributes.email` to your email field
4. Map `data.contactAttributes.plan` to custom fields for segmentation
5. Use conditional logic based on attribute values

**n8n**

In your n8n workflow:

1. Add the Formbricks trigger node
2. Access contact attributes using: `{{ $json.data.contactAttributes.email }}`
3. Use IF nodes to create conditional logic based on attributes
4. Pass attributes to downstream services (CRM, email, analytics)

**Custom Webhooks**

If you're building a custom webhook receiver, parse the JSON payload and access:

```javascript
// Node.js example
app.post('/webhook', (req, res) => {
  const payload = req.body[0]; // Formbricks sends an array
  const response = payload.data;

  // Access contact information
  if (response.contactAttributes) {
    const email = response.contactAttributes.email;
    const plan = response.contactAttributes.plan;
    const name = response.contactAttributes.name;

    // Use the attributes in your logic
    if (plan === 'Pro') {
      // Handle Pro users differently
    }
  }

  res.status(200).send('OK');
});
```

## Testing Your Webhook

To test your webhook integration with contact attributes:

1. **Create a Test Contact**
   - In Formbricks, identify a test user with attributes
   - Set attributes like email, plan, or custom fields

2. **Submit a Test Response**
   - Fill out your survey as the identified user
   - This will trigger the webhook with contact attributes

3. **Verify the Payload**
   - Check that `contactAttributes` field is populated
   - Confirm all expected attributes are present
   - Validate the data types and values

4. **Test Different Scenarios**
   - Submit responses with different attribute values
   - Test anonymous responses (contactAttributes will be null)
   - Verify conditional logic based on attributes works correctly

<Note>
  If `contactAttributes` is `null` when you expect it to have data, make sure:
  - The user is properly identified before taking the survey
  - Contact attributes have been set for that user
  - The survey allows user identification (not a completely anonymous link survey)
</Note>
