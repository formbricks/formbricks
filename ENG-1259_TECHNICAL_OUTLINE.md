# ENG-1259: Brand Kit - Custom Brand Kit with Image Processing & Authorization

## Problem Statement

Currently, users can create brand kits with manual color and typography input. To unlock brand kit value, we need:
1. **Image-based Brand Kit creation** - Upload brand assets and auto-extract colors
2. **Streamlined authorization model** - Support for team-based and granular workspace access
3. **Survey-level customization** - Allow surveys to override organization-wide brand kits with proper permission checks

This scope focuses on **image processing** and **authorization complexity**.

## Goal

Enable customers to upload brand assets (logos, images) and automatically extract color palettes, while maintaining strict authorization controls across multi-tenant and team scenarios.

### Phase 2B: Authorization Model (CRITICAL)

#### Current State (Scope 1)
```
Organization Owner    → Can CRUD all style guides
Organization Member   → Can READ style guides
Workspace Owner       → Can ACTIVATE/DEACTIVATE for workspace
Workspace Member      → Can READ style guides (if activated)
```

#### Problem: Insufficient Granularity

**Missing Scenarios:**
1. **Team-scoped Brand Kits** - A product team should be able to create style guides for their workspaces only
2. **Shared vs Private Guides** - Some guides for org, some for specific teams
3. **Survey-level Overrides** - Survey creator should be able to customize without org-wide changes
4. **API Access** - API keys need style guide access patterns (currently missing)
5. **Cross-workspace Sharing** - Teams managing multiple workspaces need consistent branding

| **Created by** | Org Owner | Team Admin | Workspace Owner | Survey Creator |

##### Permission Check Functions (Core Logic)

```typescript
// Core authorization function
async function canUserAccessStyleGuide(
  userId: string,
  styleGuideId: string,
  action: "read" | "edit" | "delete" | "share" | "activate"
): Promise<boolean> {
  // 1. Get user's roles in all scopes
  const [orgRole, teamRoles, workspaceRoles] = await Promise.all([
    getUserOrganizationRole(userId, styleGuide.organizationId),
    getUserTeamRoles(userId, styleGuide.organizationId),
    getUserWorkspaceRoles(userId),
  ]);

  // 2. Determine if user can access based on scope
  switch (styleGuide.scope) {
    case "ORGANIZATION":
      return validateOrgLevelAccess(orgRole, action);
    
    case "TEAM":
      return validateTeamLevelAccess(userId, styleGuide.teamId, action, teamRoles);
    
    case "WORKSPACE":
      return validateWorkspaceLevelAccess(userId, styleGuide.workspaceId, action, workspaceRoles);
    
    case "SURVEY":
      return validateSurveyLevelAccess(userId, styleGuide.surveyId, action);
  }
}

// Scope-specific validators
function validateOrgLevelAccess(role: OrganizationRole, action: string): boolean {
  const matrix = {
    owner: ["read", "edit", "delete", "share", "activate"],
    manager: ["read", "edit", "share"],
    member: ["read"],
    billing: ["read"],
  };
  return matrix[role]?.includes(action) ?? false;
}

function validateTeamLevelAccess(userId: string, teamId: string, action: string, teamRoles): boolean {
  const userTeamRole = teamRoles[teamId];
  const matrix = {
    admin: ["read", "edit", "delete", "share", "activate"],
    contributor: ["read", "edit"],
    viewer: ["read"],
  };
  return matrix[userTeamRole]?.includes(action) ?? false;
}

function validateWorkspaceLevelAccess(userId: string, workspaceId: string, action: string, workspaceRoles): boolean {
  const userWorkspaceRole = workspaceRoles[workspaceId];
  const matrix = {
    owner: ["read", "edit", "delete", "activate"],
    editor: ["read", "edit", "activate"],
    member: ["read"],
  };
  return matrix[userWorkspaceRole]?.includes(action) ?? false;
}

function validateSurveyLevelAccess(userId: string, surveyId: string, action: string): boolean {
  // Only survey creator can customize
  const survey = await prisma.survey.findUnique({ 
    where: { id: surveyId },
    select: { createdBy: true }
  });
  
  if (action === "edit" || action === "delete") {
    return survey?.createdBy === userId;
  }
  return action === "read"; // Anyone can read
}
```

##### Query Layer (Data Isolation)

```typescript
// Get all style guides user can access (filtered by scope + permissions)
async function getUserAccessibleStyleGuides(userId: string, organizationId: string) {
  const [orgRole, teamRoles, workspaceRoles] = await getUserRoles(userId);
  
  // Build permission-based query
  const orConditions = [];
  
  // Org-level guides (if member)
  if (orgRole && ["owner", "manager", "member"].includes(orgRole)) {
    orConditions.push({
      scope: "ORGANIZATION",
      organizationId,
    });
  }
  
  // Team-scoped guides (if in team)
  if (Object.keys(teamRoles).length > 0) {
    orConditions.push({
      scope: "TEAM",
      teamId: { in: Object.keys(teamRoles) },
      organizationId,
    });
  }
  
  // Workspace-scoped guides (if in workspace)
  if (Object.keys(workspaceRoles).length > 0) {
    orConditions.push({
      scope: "WORKSPACE",
      workspaceId: { in: Object.keys(workspaceRoles) },
    });
  }
  
  return prisma.styleGuide.findMany({
    where: { OR: orConditions },
    orderBy: { createdAt: "desc" },
  });
}
```

##### API Authorization Middleware

```typescript
// New middleware for API endpoints
export async function validateStyleGuideAccess(
  req: NextRequest,
  styleGuideId: string,
  action: "read" | "edit" | "delete"
) {
  const session = await auth();
  if (!session?.user?.id) throw new Error("Unauthorized");
  
  const styleGuide = await prisma.styleGuide.findUnique({
    where: { id: styleGuideId },
    include: { survey: { select: { workspaceId: true } } },
  });
  
  if (!styleGuide) throw new Error("Not Found");
  
  // Log access attempt
  await prisma.styleGuideAccessLog.create({
    data: {
      styleGuideId,
      userId: session.user.id,
      action,
      resourceScope: styleGuide.workspaceId || styleGuide.teamId || styleGuide.organizationId,
      ipAddress: req.headers.get("x-forwarded-for"),
    },
  });
  
  const hasAccess = await canUserAccessStyleGuide(
    session.user.id,
    styleGuideId,
    action
  );
  
  if (!hasAccess) throw new Error("Forbidden");
  return styleGuide;
}
```

## Risk Assessment

### High Risk: Authorization Complexity
- **Risk**: Data leakage across teams/workspaces
- **Mitigation**: Query-layer isolation + access logging + regular audits

### Medium Risk: Performance (Multi-scope queries)
- **Risk**: Slow permission checks for users in many teams
- **Mitigation**: Cache user role matrix for 5min, add indexes on scope/userId

### Medium Risk: Backwards Compatibility
- **Risk**: Existing org-wide guides must continue working
- **Mitigation**: Default `scope=ORGANIZATION` for existing records, migration script
